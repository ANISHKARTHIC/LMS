# Virtual Lab Evaluation System

Production-ready Django web application for evaluating virtual lab experiment submissions with AI-assisted scoring, admin review controls, and certificate generation.

## Features

- Admin panel for managing experiments
- Student submission flow (name, roll number, Tinkercad URL, screenshot, optional explanation)
- AI evaluation of screenshot and explanation
- Weighted final scoring
  - Screenshot AI: 50%
  - Link validity: 20%
  - Explanation AI: 10%
  - Admin review: 20%
- Pass/fail automation based on experiment pass marks
- Admin override score and approve/reject workflow
- PDF certificate download for passed submissions
- Submission history page
- Staff analytics dashboard (pass rate + experiment breakdown)

## Tech Stack

- Django 6
- SQLite (default) and PostgreSQL via `DATABASE_URL`
- Gemini API (free tier) with optional OpenAI fallback
- ReportLab for certificate PDFs
- WhiteNoise + Gunicorn for deployment

## Local Setup

1. Create/activate virtual environment from repository root:

```bash
python -m venv venv
source venv/bin/activate
```

2. Install dependencies:

```bash
pip install -r virtual_lab_eval/requirements.txt
```

3. Create environment file:

```bash
cp virtual_lab_eval/.env.example virtual_lab_eval/.env
```

4. Run migrations:

```bash
python manage.py makemigrations users experiments submissions
python manage.py migrate
```

5. Create admin user:

```bash
python manage.py create_default_admin
```

6. Start development server:

```bash
python manage.py runserver
```

7. Open in browser:

- Student UI: `http://127.0.0.1:8000/`
- Admin Dashboard UI: `http://127.0.0.1:8000/admin-panel/`
- Django Admin: `http://127.0.0.1:8000/admin/`

Default local credentials:

- Username: `admin`
- Password: `admin123`

## Free API Configuration (Default)

The project uses free-tier Gemini API by default.

1. Create a free API key in Google AI Studio.
2. Set these in `.env`:

```env
AI_PROVIDER=gemini
GEMINI_API_KEY=your_key_here
GEMINI_VISION_MODEL=gemini-2.0-flash
GEMINI_TEXT_MODEL=gemini-2.0-flash
```

Optional: switch to OpenAI by setting:

```env
AI_PROVIDER=openai
OPENAI_API_KEY=your_openai_key
```

If key is missing, submission still saves and shows a clear message that AI scoring was skipped.

If the free-tier provider returns rate limit (HTTP 429):

- The app applies a provisional screenshot score and avoids exposing raw API errors.
- You can re-evaluate later from Django Admin action: `Re-evaluate selected submissions with AI`.
- The app uses a single Gemini call per submission (image + explanation) to reduce quota pressure.
- It retries with backoff and can fall back to `GEMINI_FALLBACK_MODEL`.
- You can normalize old 429 records with:

```bash
python manage.py normalize_rate_limited_submissions
```

If you see HTTP 404 from Gemini model endpoint:

- Your configured model is not available for your API key/account/region.
- Set `GEMINI_FALLBACK_MODEL=` (empty) or use a verified model from Google AI Studio.
- Re-run AI scoring from admin action: `Re-evaluate selected submissions with AI`.

## Deployment

### Render (Blueprint Deploy)

This repository now includes root-level deployment files:

- `render.yaml`
- `Procfile`
- `runtime.txt`

Steps:

1. Push this repository to GitHub/GitLab.
2. In Render, choose **New +** -> **Blueprint** and select this repository.
3. Render will read `render.yaml` and create:
  - one web service
  - one PostgreSQL database
4. Set secrets in Render dashboard:
  - `GEMINI_API_KEY`
  - `DEFAULT_ADMIN_PASSWORD`
5. Deploy.

The pre-deploy command runs automatically:

```bash
python manage.py migrate && python manage.py collectstatic --noinput && python manage.py create_default_admin
```

After deploy:

- App URL: generated by Render
- Admin dashboard: `/admin-panel/`
- Django admin: `/admin/`

### Heroku / Other PaaS (Procfile mode)

1. Ensure app uses root `Procfile`.
2. Install dependencies from root requirements:

```bash
pip install -r requirements.txt
```

3. Set env vars:

- `SECRET_KEY`
- `DEBUG=False`
- `ALLOWED_HOSTS=<your-domain>`
- `CSRF_TRUSTED_ORIGINS=https://<your-domain>`
- `DATABASE_URL=<postgres-url>`
- `AI_PROVIDER=gemini`
- `GEMINI_API_KEY=<your-key>`
- `DEFAULT_ADMIN_PASSWORD=<strong-password>`

4. Run release steps:

```bash
python manage.py migrate
python manage.py collectstatic --noinput
python manage.py create_default_admin
```
